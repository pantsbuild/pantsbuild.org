## Basic options

### `args`

<Option
  cli_repr={`--scalac-args="[<shell_str>, <shell_str>, ...]"`}
  env_repr='PANTS_SCALAC_ARGS'
  toml_repr={`[scalac]
args = [
    <shell_str>,
    <shell_str>,
    ...,
]`}
  default_repr={`[]`}
>

Global `scalac` compiler flags, e.g. `--scalac-args='-encoding UTF-8'`.

</Option>

## Advanced options

### `plugins_global`

<Option
  cli_repr={`--scalac-plugins-global="['<str>', '<str>', ...]"`}
  env_repr='PANTS_SCALAC_PLUGINS_GLOBAL'
  toml_repr={`[scalac]
plugins_global = [
    '<str>',
    '<str>',
    ...,
]`}
  default_repr={`[]`}
>

A list of addresses of `scalac_plugin` targets which should be used for compilation of all Scala targets in a build.

If you set this, you must also set `[scalac].plugins_global_lockfile`.

</Option>

### `plugins_global_lockfile`

<Option
  cli_repr={`--scalac-plugins-global-lockfile=<str>`}
  env_repr='PANTS_SCALAC_PLUGINS_GLOBAL_LOCKFILE'
  toml_repr={`[scalac]
plugins_global_lockfile = <str>`}
  default_repr={`<default>`}
>

The filename of the lockfile for global plugins. You must set this option to a file path, e.g. &#x27;3rdparty/jvm/global_scalac_plugins.lock&#x27;, if you set `[scalac].plugins_global`.

</Option>
