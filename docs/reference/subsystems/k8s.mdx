## Basic options

### `available_contexts`

<Option
  cli_repr={`--k8s-available-contexts="['<str>', '<str>', ...]"`}
  env_repr='PANTS_K8S_AVAILABLE_CONTEXTS'
  toml_repr={`[k8s]
available_contexts = [
    '<str>',
    '<str>',
    ...,
]`}
  default_repr={`[]`}
>

List of available contexts for `kubectl` command. `k8s_bundle` context will be validated against this list.

You have to explicitly provide the list, because it will be shared with people using the pants repo. We can&#x27;t parse the KUBECONFIG env var because different people might have different private clusters, e.g. minikube or kind, which means pants validation will give different results.

</Option>
