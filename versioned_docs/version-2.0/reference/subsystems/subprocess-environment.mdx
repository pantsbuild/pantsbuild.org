Environment settings for forked subprocesses.

Backend: <span className="color--primary">`pants.core`</span>

Config section: <span className="color--primary">`[subprocess-environment]`</span>

<span className={styles.options}>

## Basic options

None

## Advanced options

### `env_vars`

<Option
  cli_repr={`--subprocess-environment-env-vars="['<str>', '<str>', ...]"`}
  env_repr='PANTS_SUBPROCESS_ENVIRONMENT_ENV_VARS'
  default_repr={`[\n  "LANG",\n  "LC_CTYPE",\n  "LC_ALL"\n]`}
>

Environment variables to set for process invocations. Entries are either strings in the form `ENV_VAR=value` to set an explicit value; or just `ENV_VAR` to copy the value from Pants&#x27;s own environment.

Each ENV_VAR must be one of `LANG`, `LC_CTYPE`, `LC_ALL`, `http_proxy`, `https_proxy`, `ftp_proxy`, `all_proxy`, `no_proxy`, `HTTP_PROXY`, `HTTPS_PROXY`, `FTP_PROXY`, `ALL_PROXY`, `NO_PROXY`, `REQUESTS_CA_BUNDLE`.

</Option>
