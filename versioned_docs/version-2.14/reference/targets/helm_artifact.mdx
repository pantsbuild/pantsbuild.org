A third party Helm artifact.

Backend: <span className="color--primary">`pants.backend.experimental.helm`</span>

---

<span className={styles.options}>

## `artifact`

<Field
    type_repr={`str`}
>

Artifact name of the chart, without version number.

</Field>

## `version`

<Field
    type_repr={`str`}
>

The `version` part of a third party Helm chart.

</Field>

## `description`

<Field
    type_repr={`str | None`}
    default_repr={`None`}
>

A human-readable description of the target.

Use `/home/josh/work/scie-pants/dist/pants list --documented ::` to see all targets with descriptions.

</Field>

## `registry`

<Field
    type_repr={`str | None`}
    default_repr={`None`}
>

Either registry alias (prefixed by `@`) configured in `[helm.registries]` for the Helm artifact or the full OCI registry URL.

</Field>

## `repository`

<Field
    type_repr={`str | None`}
    default_repr={`None`}
>

Either a HTTP(S) URL to a classic repository, or a path inside an OCI registry (when `registry` is provided).

</Field>

## `tags`

<Field
    type_repr={`Iterable[str] | None`}
    default_repr={`None`}
>

Arbitrary strings to describe a target.

For example, you may tag some test targets with &#x27;integration_test&#x27; so that you could run `/home/josh/work/scie-pants/dist/pants --tag='integration_test' test ::` to only run on targets with that tag.

</Field>

</span>
