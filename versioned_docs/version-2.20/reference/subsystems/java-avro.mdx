## Advanced options

### `artifacts`

<Option
  cli_repr={`--java-avro-artifacts="['<str>', '<str>', ...]"`}
  env_repr='PANTS_JAVA_AVRO_ARTIFACTS'
  toml_repr={`[java-avro]
artifacts = [
    '<str>',
    '<str>',
    ...,
]`}
  default_repr={`[\n  "org.apache.avro:avro-tools:{version}"\n]`}
>

Artifact requirements for this tool using specified as either the address of a `jvm_artifact` target or, alternatively, as a colon-separated Maven coordinates (e.g., `group:name:version`). For Maven coordinates, the string `{version}` version will be substituted with the value of the `[java-avro].version` option.

</Option>

### `jvm_options`

<Option
  cli_repr={`--java-avro-jvm-options="['<str>', '<str>', ...]"`}
  env_repr='PANTS_JAVA_AVRO_JVM_OPTIONS'
  toml_repr={`[java-avro]
jvm_options = [
    '<str>',
    '<str>',
    ...,
]`}
  default_repr={`[]`}
>

List of JVM options to pass to `java-avro` JVM processes.

Options set here will be added to those set in `[jvm].global_options`. Please check the documentation for the `jvm` subsystem to see what values are accepted here.

</Option>

### `lockfile`

<Option
  cli_repr={`--java-avro-lockfile=<str>`}
  env_repr='PANTS_JAVA_AVRO_LOCKFILE'
  toml_repr={`[java-avro]
lockfile = <str>`}
  default_repr={`<default>`}
>

Path to a lockfile used for installing the tool.

Set to the string `<default>` to use a lockfile provided by Pants, so long as you have not changed the `--version` option. See https://github.com/pantsbuild/pants/blob/release_2.20.4rc0/src/python/pants/backend/codegen/avro/java/avro-tools.default.lockfile.txt for the default lockfile contents.

To use a custom lockfile, set this option to a file path relative to the build root, then run `pants jvm-generate-lockfiles --resolve=java-avro`.

</Option>

### `version`

<Option
  cli_repr={`--java-avro-version=<str>`}
  env_repr='PANTS_JAVA_AVRO_VERSION'
  toml_repr={`[java-avro]
version = <str>`}
  default_repr={`1.11.0`}
>

Version string for the tool. This is available for substitution in the `[java-avro].artifacts` option by including the string `{version}`.

</Option>
