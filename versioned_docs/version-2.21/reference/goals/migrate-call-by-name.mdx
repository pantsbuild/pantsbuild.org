Migrate from `Get` syntax to call-by-name syntax (#19730). This is a **destructive** operation, so only run this on source controlled files that you are prepared to revert if necessary.

This goal will attempt to migrate the set of paths/targets specified at the command line if they are part of the &#x22;migration plan&#x22;. This migration does not add any new files, but instead modifies existing files in-place without any formatting. The resulting changes should be reviewed, tested, and formatted/linted before committing.

The migration plan is a JSON representation of the rule graph, which is generated by the engine based on the active backends/rules in the project.

Each item in the migration plan is a rule that contains the old `Get` syntax, the associated input/output types, and the new function to directly call. The migration plan can be dumped as JSON using the `--json` flag, which can be useful for debugging. For example:

&#123;
&#x22;filepath&#x22;: &#x22;src/python/pants/source/source_root.py&#x22;,
&#x22;function&#x22;: &#x22;get_source_roots&#x22;,
&#x22;gets&#x22;: [&#123;
&#x22;input_types&#x22;: [&#123; &#x22;module&#x22;: &#x22;pants.source.source_root&#x22;, &#x22;name&#x22;: &#x22;SourceRootsRequest&#x22; &#125;],
&#x22;output_type&#x22;: &#123; &#x22;module&#x22;: &#x22;pants.source.source_root&#x22;, &#x22;name&#x22;: &#x22;OptionalSourceRootsResult&#x22; &#125;,
&#x22;rule_dep&#x22;: &#123; &#x22;function&#x22;: &#x22;get_optional_source_roots&#x22;, &#x22;module&#x22;: &#x22;pants.source.source_root&#x22; &#125;
&#125;],
